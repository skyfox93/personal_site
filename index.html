Feature Ownership

    When I am building a feature, I ask myself the following questions:
  • Is the code structured in a way that is transparent and readable for the rest of the team?
  • Am I returning the data in a structure and format which is easily consumable and consistent with the way our api normally returns data?
  • Are we fetching data efficiently? Are we fetching only the data we need, without unnecessary complexity?
  • Are we handling edge cases correctly and surfacing errors correctly?
  • Are there UX hiccups that are not mentioned in the product requirements? For example, are we missing a spinner or visual feedback while the page is fetching data? What happens when the user resizes the browser window? Small details like this may not be part of the written requirements.
